Setting things up

1. Check whether Git is present ...

git version

2. Set up your local Git installation.

git config --global user.name "John Doe"
git config --global user.email johndoe@example.com
git config --global -l --show-origin

3. List the contents (for kicks & giggles).

cat C:/Users/johndoe/.gitconfig

Personalization

1. Set up diff & merge tool

git config --global diff.tool bc
git config --global difftool.bc.path ".../BComp.exe"
git config --global merge.tool bc
git config --global mergetool.bc.path ".../BComp.exe"

2. Set up an editor for Git's use

git config --global core.editor "code --wait"

Getting a sample project

git remote

Working with files [1/2]

1. Add files (to commit)

git add/rm/mv <file>
git add -p
git add -i
git rm --cached <file> 
git reset HEAD <file>
git reset HEAD^ <file>
git reset HEAD^^ <file>
git reset HEAD <file> --hard
git reset HEAD~ <file> --hard
git reset HEAD~~ <file> --hard
git checkout -- <file>

2. Find out the local file status

git status
git status -s

Working with files [2/2]

3. Check the differences (before the commit)

git diff
git diff --staged

4. Check which files are being ignored

cat .gitignore

Commits, finally! [1/2]

1. Making an actual commit

git commit
git commit -m <message>
git commit -a -m <message>
git show <sha-1>

2. Browsing the history (output)

git log
git log --grep
git log --pretty=oneline|short
git log --decorate
git log --graph
git log -p -n 3
git log --stat

Commits, finally! [2/2]

3. Browsing the history (filtering)

git log --since=2.weeks
git log --since="2017-05-01" --until="2017-05-10"
git log -S snafu
git log --grep Personalization
git log --author="Sebastian"
git log -u index.html
git log <branch>..HEAD
git log ^<branch1>..<branch2>
git log <branch1>...<branch2>

4. Fixing last commit

git commit -m "Feature X"
<do something>
git commit --amend

git branch -vv

2. Merging 101

git merge fugazi
